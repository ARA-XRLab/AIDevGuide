  타이머를 추가했는데도 여전히 증가하는 이유:

  핵심 문제: SetActive(false)는 비활성화만 하고 액터를 제거하지 않음

  OnLifeTimeExpired() (EnemyBulletActor.cpp:103-106):
  void AEnemyBulletActor::OnLifeTimeExpired()
  {
      SetActive(false);  // 비활성화만 함, Destroy() 호출 안 함
  }

  SetActive(false)가 하는 일:
  - 메시 숨김
  - 충돌 비활성화
  - 액터는 여전히 월드에 존재 ← 문제!

  구조적 차이

  PlayerBulletActor:
  - 오브젝트 풀링 사용
  - SetActive(false) → 풀에 반환 → 재사용
  - 10개로 일정하게 유지 ✅

  EnemyBulletActor:
  - 오브젝트 풀링 없음
  - 매번 새로 SpawnActor()
  - SetActive(false)만 호출 → 비활성화된 채 월드에 쌓임
  - Destroy() 호출 안 함 → 계속 증가 ❌

  적이 총알을 어떻게 생성하는지 확인하겠습니다.

  문제의 흐름

  1. 총알 발사 → 매번 새로운 액터를 SpawnActor()로 생성
  2. 5초 후 → 타이머가 OnLifeTimeExpired() 호출
  3. OnLifeTimeExpired() → SetActive(false) 호출 (비활성화만)
  4. 액터는 여전히 월드에 존재 → Destroy() 호출 안 함
  5. ActorTracking → 비활성화된 총알도 모두 카운트
  6. 결과 → 계속 증가 📈

  ✅ 해결책

  OnLifeTimeExpired()에서 Destroy()를 호출해야 합니다:

  void AEnemyBulletActor::OnLifeTimeExpired()
  {
      Destroy();  // SetActive(false) 대신 액터를 완전히 제거
  }

  이렇게 수정하면 5초 후 총알이 완전히 제거되어 메모리 누수가 해결됩니다.
