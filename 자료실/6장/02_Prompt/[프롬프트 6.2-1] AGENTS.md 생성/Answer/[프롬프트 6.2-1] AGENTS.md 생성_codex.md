# CODEX

## 프로젝트 요약
- 엔진 버전: Unreal Engine 5.7 (`SpaceShooter.uproject`의 EngineAssociation)
- 모듈 구조: 단일 런타임 모듈 `SpaceShooter` (의존: Core, CoreUObject, Engine, InputCore, EnhancedInput, UMG)
- 핵심 플레이: 플레이어가 Y/Z 평면으로 이동하며 연속 사격, 적은 스폰 포인트에서 생성되어 접근 후 버스트 사격, 충돌 시 피해 및 폭발 이펙트

## 주요 경로
- 프로젝트 파일: `SpaceShooter.uproject`
- 소스 코드: `Source/SpaceShooter`
- 주요 에셋 루트: `Content/Shooting` (코드에서 `/Game/Shooting/...` 경로 사용)
- UI 위젯: `Content/Shooting` 하위 UMG 에셋 (코드에서 `UMainWidget`, `UGameOverWidget` 사용)

## 런타임 흐름 (GameMode → PlayerPawn → EnemyManager → Enemy)
1. `AShootingGameMode::BeginPlay`
   - 메인 HUD 위젯 생성 및 표시
   - 입력 모드 GameOnly 설정, 마우스 커서 숨김
   - 하이스코어 로드 후 점수 표시 갱신
2. `APlayerPawn::BeginPlay`
   - Enhanced Input 매핑 컨텍스트 추가 (`IMC_Player`)
   - HP 초기화, 머티리얼 동적 인스턴스 생성
   - 플레이어 탄환 풀 생성 및 비활성화
3. `AEnemyManagerActor::BeginPlay`
   - 이름에 "SpawnPoint"가 포함된 액터 수집
   - `SpawnFrequency` 간격으로 적 스폰
4. `AEnemyActor`
   - 스폰 시 플레이어 방향으로 이동
   - `EncounterRadius` 안에 들어오면 버스트 사격 시작
   - 충돌 시 플레이어 피해 처리 및 게임오버 UI 호출

## 프로젝트 규칙
- 충돌 프로필
  - Player: `PlayerPawn`의 `BoxComp`
  - Enemy: `EnemyActor`의 `BoxComp`
  - PlayerBullet: `PlayerBulletActor`의 `BoxComp`
  - EnemyBullet: `EnemyBulletActor`의 `BoxComp`
  - DestroyZone: `DestroyZoneActor`의 `BoxComp`
- 에셋 경로 규칙
  - 콘텐츠는 `/Game/Shooting/...` 하위에 위치 (`Content/Shooting` 매핑)
  - 메시/머티리얼/사운드/이펙트는 코드에서 `ConstructorHelpers::FObjectFinder`로 직접 참조
- 네이밍 규칙 및 관례
  - 스폰 포인트는 이름에 "SpawnPoint" 포함 필요
  - 위젯 바인딩 변수명은 UMG 디자이너의 위젯 이름과 일치 필요 (`BindWidget`)
  - 클래스 네이밍은 UE 표준 접두사(`A`, `U`) 사용

## 자주 하는 작업
- 입력 추가
  - 새 `UInputAction` 생성 후 `IMC_Player`에 매핑
  - `APlayerPawn::SetupPlayerInputComponent`에서 바인딩 추가
  - 블루프린트/디폴트에서 `IMC_Player`, `IA_*` 참조 연결
- 적 타입 추가
  - `AEnemyActor` 파생 클래스 생성 및 행동 로직 작성
  - `EnemyManagerActor`의 `EnemyFactory`를 새 클래스/블루프린트로 교체
  - 필요 시 적 전용 탄환 클래스 및 충돌 프로필 추가
- UI 요소 추가
  - UMG 위젯에 새 요소 추가
  - `UMainWidget` 또는 `UGameOverWidget`에 `BindWidget` 멤버 추가
  - 점수/HP 등 갱신 함수에 업데이트 로직 추가
