# SpaceShooter - AI QA 에이전트 컨텍스트

## 프로젝트 요약

- **엔진**: Unreal Engine 5.7 (C++ / Blueprint 혼합)
- **장르**: 2.5D 종스크롤 슈팅 (SpaceShooter)
- **모듈**: `SpaceShooter` 단일 모듈 (Core, CoreUObject, Engine, InputCore, EnhancedInput, UMG 의존)
- **핵심 플레이**: 플레이어가 우주선을 조종하며 WASD 이동 + 연사 발사로 적을 격파, 점수를 획득하는 아케이드 슈팅
- **버전**: `6.1/`, `6.2/` 두 폴더에 동일한 코드 존재

## C++ 클래스 구조

| 클래스 | 부모 클래스 | 역할 |
|--------|------------|------|
| `AShootingGameMode` | AGameModeBase | 점수 관리, UI 생성, 세이브/로드, 게임오버 처리 |
| `APlayerPawn` | APawn | 플레이어 이동, 발사, HP/무적, 총알 풀링 |
| `ABulletActor` (PlayerBulletActor) | AActor | 플레이어 총알, 오브젝트 풀 대상 |
| `AEnemyActor` | AActor | 적 AI(이동→조우→버스트 사격), HP 시스템 |
| `AEnemyBulletActor` | AActor | 적 총알, 수명 타이머 |
| `AEnemyManagerActor` | AActor | 적 스폰 관리 (2초 간격, 랜덤 스폰 포인트) |

## 런타임 흐름

```
GameMode::BeginPlay()
  ├─ MainWidget 생성 및 뷰포트 추가
  ├─ GConfig에서 HighScore 로드
  └─ 입력 모드: Game Only

PlayerPawn::BeginPlay()
  ├─ Enhanced Input Mapping Context 바인딩
  ├─ HP 초기화 (MaxHP = 3)
  ├─ 총알 풀 생성 (MaxBulletCount = 10)
  └─ Dynamic Material Instance 생성 (히트 이펙트용)

EnemyManagerActor::BeginPlay()
  ├─ 레벨 내 "SpawnPoint" 이름 액터 탐색
  └─ 2초 간격 스폰 타이머 시작

[매 프레임 게임 루프]
  PlayerPawn::Tick()
    ├─ IA_Move 입력 → 위치 이동 + 메시 틸트
    └─ IA_Fire Pressed → 0.1초 반복 타이머로 GetBullet() → 발사

  EnemyActor::Tick()
    ├─ 플레이어 방향으로 이동 (Speed = 500)
    └─ EncounterRadius(500) 내 진입 시
        ├─ 이동 정지
        ├─ Y-Z 평면 발사 방향 계산
        ├─ 3~5발 버스트 사격 (0.2초 간격)
        └─ 발사 방향으로 이동 재개

[충돌 처리]
  PlayerBullet + Enemy → 적 파괴, 점수 +100, 총알 풀 반환
  EnemyBullet + Player → 플레이어 피격 (1 데미지), 무적 1초
  Enemy + Player → 플레이어 피격 (1 데미지), 적 파괴
  Any + DestroyZone → 액터 제거 (총알은 풀 반환)

[플레이어 사망]
  HP == 0 → PlayerPawn 파괴 → 게임 일시정지 → GameOverWidget 표시
```

## 충돌 프로필 규칙

5개의 커스텀 충돌 채널이 `DefaultEngine.ini`에 정의되어 있습니다:

| 채널 | Trace Channel | 용도 |
|-------|---------------|------|
| Player | ECC_GameTraceChannel1 | 플레이어 |
| Enemy | ECC_GameTraceChannel2 | 적 |
| PlayerBullet | ECC_GameTraceChannel3 | 플레이어 총알 |
| DestroyZone | ECC_GameTraceChannel4 | 제거 영역 |
| EnemyBullet | ECC_GameTraceChannel5 | 적 총알 |

**충돌 매트릭스**:
- Player ↔ Enemy: Overlap
- Player ↔ EnemyBullet: Overlap
- Enemy ↔ PlayerBullet: Overlap
- PlayerBullet ↔ DestroyZone: Overlap
- EnemyBullet ↔ DestroyZone: Overlap
- 그 외 모든 조합: Ignore (WorldStatic, WorldDynamic, Visibility, Camera 등)

## 프로젝트 규칙 및 패턴

### 네이밍 규칙
- C++ 클래스: `A` 접두사 (Actor), `U` 접두사 (UObject/Widget)
- 블루프린트: `BP_` 접두사 (액터), `WBP_` 접두사 (위젯)
- 입력 액션: `IA_` 접두사, 매핑 컨텍스트: `IMC_` 접두사
- 머티리얼 인스턴스: `MI_` 접두사, 마스터 머티리얼: `M_` 접두사
- 파티클: `P_` 접두사

### 에셋 경로 규칙
- 모든 게임 에셋: `/Game/Shooting/` 하위
- 블루프린트는 C++ 클래스를 상속하여 메시, 머티리얼 등 에셋 참조를 설정

### 핵심 디자인 패턴
- **오브젝트 풀링**: 플레이어 총알만 풀링 적용 (`SetActive(true/false)`)
- **타이머 기반 로직**: 발사 속도, 스폰, 무적 시간, 블링크 이펙트 모두 `FTimerHandle` 사용
- **Dynamic Material Instance**: 플레이어 히트 플래시 (`HitIntensity` 파라미터)
- **세이브 시스템**: `GConfig` (INI 기반), `GGameUserSettingsIni`에 HighScore 저장

## 자주 하는 작업

### 새 입력 추가
1. `Content/Shooting/Input/`에 새 Input Action 에셋 생성 (예: `IA_Bomb`)
2. `IMC_Player` 매핑 컨텍스트에 키 바인딩 추가
3. `PlayerPawn.h`에 콜백 함수 선언
4. `PlayerPawn.cpp`의 `SetupPlayerInputComponent()`에서 `BindAction()` 호출
5. 콜백 함수 구현

### 새 적 타입 추가
1. `AEnemyActor`를 상속하는 새 C++ 클래스 생성 (또는 기존 클래스 확장)
2. `Public/`에 헤더, `Private/`에 구현 파일 배치
3. 이동 패턴, 발사 로직, HP를 오버라이드 또는 파라미터화
4. 블루프린트(`BP_NewEnemy`)를 생성하여 메시/머티리얼 할당
5. `EnemyManagerActor`에 새 적 타입 스폰 로직 추가
6. 필요 시 `DefaultEngine.ini`에 충돌 프로필 추가

### UI 요소 추가
1. `UUserWidget`을 상속하는 C++ 클래스 생성 (`Public/`, `Private/`)
2. `UPROPERTY(meta=(BindWidget))`으로 위젯 바인딩 선언
3. `Content/Shooting/Blueprints/`에 Widget Blueprint 생성 (`WBP_` 접두사)
4. 블루프린트에서 C++과 동일한 이름의 위젯 배치
5. `ShootingGameMode`에서 위젯 생성 및 뷰포트 추가

## 알려진 이슈 및 개선점

1. **적 총알 수명 타이머 미구현**: `EnemyBulletActor`에 타이머 핸들이 선언되어 있으나 실제 타이머가 설정되지 않음
2. **적 HP 미활용**: `EnemyActor`에 HP 변수가 있으나 항상 1히트에 파괴됨
3. **플레이어 이동 범위 미제한**: 화면 밖으로 이동 가능
4. **적 풀링 미적용**: 적은 매번 Spawn/Destroy → GC 부하 가능성
5. **난이도 스케일링 없음**: 시간 경과에 따른 난이도 증가 로직 없음
6. **TryStartFiring 가드 제거**: 다중 타이머 축적 가능한 잠재적 버그
7. **총알 풀 최대 크기 미제한**: 수요 초과 시 풀이 무한 증가 가능

## AI 에이전트의 책임과 한계

### 할 수 있는 것
- **코드 분석**: C++ 소스, 블루프린트 구조, 설정 파일 정적 분석
- **이슈 탐지**: 잠재적 버그, 메모리 누수, 성능 병목, 충돌 프로필 불일치 식별
- **수정 제안**: 코드 변경 방안, 리팩토링 방향, 최적화 전략 제시
- **패턴 식별**: 코딩 규칙 위반, 안티패턴, 일관성 결여 탐지
- **문서화**: 코드 흐름 정리, 클래스 관계도 작성, QA 리포트 생성
- **프로파일링 가이드**: Unreal Insights 마커 삽입 위치 및 방법 제안

### 할 수 없는 것
- **최종 의사결정**: 아키텍처 변경, 기능 추가/제거 결정은 개발자의 권한
- **프로덕션 배포**: 빌드, 패키징, 배포 프로세스 직접 수행 불가
- **게임 밸런스 판단**: 적 스폰 주기, 총알 속도, HP 수치 등 밸런싱은 게임 디자이너 영역
- **런타임 테스트**: 실제 에디터/게임 실행을 통한 동적 테스트 불가
- **에셋 검증**: 메시, 텍스처, 사운드 등 바이너리 에셋의 품질 확인 불가

### 개발자 확인 필수 사항
- **모든 코드 수정**: AI가 제안한 코드 변경은 반드시 개발자가 리뷰 후 적용
- **성능 영향**: 오브젝트 풀링 변경, 타이머 주기 수정 등 성능에 영향을 주는 변경
- **게임플레이 변경**: 이동 속도, 발사 패턴, 충돌 로직 등 플레이 경험에 영향을 주는 수정
- **충돌 프로필 수정**: 새 채널 추가 또는 기존 매트릭스 변경 시
- **세이브 시스템 변경**: 저장 데이터 구조 변경 시 하위 호환성 확인
- **UI 레이아웃 변경**: 위젯 바인딩 이름 변경 시 블루프린트 동기화 필요
