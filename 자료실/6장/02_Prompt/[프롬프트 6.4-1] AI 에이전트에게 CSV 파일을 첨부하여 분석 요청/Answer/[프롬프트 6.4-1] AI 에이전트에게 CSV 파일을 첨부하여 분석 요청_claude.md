  🔴 CRITICAL — 즉시 수정 필요

  1. FTimerManager_Tick — 최대 1,436ms 스파이크

  FTimerManager_Tick: 265 calls, I.Avg=10.8ms, I.Max=1,436ms
  UWorld_Tick:        I.Avg=32.7ms, I.Max=1,477ms

  원인 추정: 타이머 콜백(Blueprint SetTimer)에서 무거운 연산 실행. 특히 스파이크가 1.4초면 타이머 콜백 안에서 동기 에셋 로드(LoadObject,
  FlushAsyncLoading) 또는 대량 오브젝트 생성이 일어나고 있을 가능성이 높음.

  확인 방법: Blueprint에서 Set Timer by Function Name / Event 사용 시 콜백 로직 검토.

  수정 방법:
  - 타이머 콜백에서 무거운 연산 제거
  - LoadObject / Spawn 계열 작업은 타이머 대신 AsyncLoadAsset 사용
  - 연산이 길 경우 여러 프레임에 걸쳐 분산 처리 (코루틴 패턴)

  ---
  2. USceneComponent::UpdateOverlaps — 476,341 calls (프레임당 ~1,800회)

  USceneComponent::UpdateOverlaps: 476,341 calls, Total=1.86s
  OnBoxCompOverlap:                3,298 calls, I.Max=3.77ms

  원인 추정: 발사체(Bullet)나 적 오브젝트가 이동할 때마다 겹침(Overlap) 계산이 트리거됨. SpaceShooter 특성상 총알이 많으면 프레임당 수천
  번의 Overlap 체크가 발생.

  수정 방법:
  // 발사체 컴포넌트에서
  CollisionComponent->SetGenerateOverlapEvents(false); // 필요할 때만 true
  // 또는 ProjectSettings > Collision에서 발사체 채널 최적화
  - 발사체는 Overlap 대신 LineTrace / SphereTrace 사용 고려
  - bGenerateOverlapEvents = false 기본값으로 설정, 충돌 필요한 것만 활성화
  - Collision Preset을 Custom으로 설정하여 불필요한 채널 체크 제거

  🟠 MAJOR — 성능에 큰 영향

  3. GPU — Lumen + VSM이 SpaceShooter에 과도함

  SceneRender:           I.Avg=13.5ms, I.Max=269ms (!)
  RenderDeferredLighting: I.Avg=1.92ms
  ShadowDepths (VSM):    I.Avg=1.62ms, I.Max=3.85ms
  LumenScreenProbeGather: I.Avg=1.03ms
  UpdateGlobalDistanceField: Total=0.263s

  GPU 렌더링 자체는 평균 19ms로 양호하지만 최대 269ms 스파이크가 존재. SpaceShooter 장르에서 Lumen + Virtual Shadow Maps는 오버스펙일 수
  있음.

  수정 방법:
  - r.Lumen.Reflections.Allow 0 — 반사 Lumen 비활성화
  - r.Lumen.GlobalIllumination.Allow 0 — GI Lumen 비활성화, Screen Space GI로 대체
  - VSM → r.Shadow.Virtual.Enable 0, 기존 Cascaded Shadow Maps 사용
  - Project Settings > Rendering에서 적절한 품질로 낮춤

  4. FPrimitiveSceneInfo_CacheMeshDrawCommand — 4,536 calls

  CacheMeshDrawCommand: 4,536 calls
  AddStaticMeshes:      4,536 calls

  매 프레임 대량의 새 Primitive가 씬에 추가됨 → 총알/이펙트 Spawn 시 오브젝트 풀링 없이 직접 Spawn하고 있는 것으로 추정.

  수정 방법: Object Pooling 구현 — 총알, 이펙트, 적을 매번 SpawnActor 하지 말고 풀에서 재사용.

  GameThread에서 렌더링 업데이트가 몰리는 구간 존재. Transform 업데이트를 여러 프레임에 분산 필요.

  7. Slate::DrawWindows — I.Max=59ms (UI 스파이크)

  Slate::DrawWindows:  I.Max=59ms (평균 0.68ms)
  Slate::Tick (Time and Widgets): I.Max=59ms

  UI 위젯 중 특정 순간에 급격히 무거워지는 경우 있음. WBP_Main 위젯의 복잡한 Tick 또는 갑작스러운 레이아웃 재계산이 원인일 가능성.

  수정 방법:
  - WBP_Main에서 Tick 비활성화하고 이벤트 기반으로 변경
  - 자주 변경되지 않는 위젯은 Invalidation Box로 묶기

  8. Windows_LoadLibrary — 408 calls, I.Max=7.7ms (게임플레이 중 DLL 로딩)

  Windows_LoadLibrary: 408 calls, Total=0.413s
  ResolveMissingImports: 393 calls, I.Max=12.3ms

  게임플레이 중에 DLL이 동적으로 로드되고 있음 → 첫 에셋 사용 시 셰이더/모듈 초기화가 발생하는 것으로 추정.

  수정 방법: PIE(에디터) 환경에서 발생하는 현상일 수 있으므로 패키지 빌드 후 재측정 권장.
