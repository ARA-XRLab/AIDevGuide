● HP UI 초기화 문제 분석 결과                                                                                                                        
               
  ---                                                                                                                                                
  근본 원인 (Root Cause)          
                                                                                                                                                     
  1. 변수 초기화 상태                                                                                                                                
  - PlayerPawn.h:103 - int32 CurHP = 0; ← 멤버 변수 초기값이 0
  - PlayerPawn.h:83 - int32 MaxHP = 3; ← MaxHP는 3으로 설정

  2. BeginPlay() 호출 순서 문제

  void APlayerPawn::BeginPlay()
  {
      Super::BeginPlay();

      // 77-80행: UI 업데이트 (CurHP = 0 상태)
      if (AShootingGameMode* GameMode = Cast<AShootingGameMode>(GetWorld()->GetAuthGameMode()))
      {
          GameMode->UpdatePlayerHP(CurHP);  // ← CurHP = 0을 전달
      }

      // 82행: HP 초기화 (UI 업데이트 후)
      CurHP = MaxHP;  // ← 이제야 CurHP = 3으로 설정
  }

  3. 호출 체인
  APlayerPawn::BeginPlay()
    ├─ [79행] GameMode->UpdatePlayerHP(0)  ← CurHP = 0 전달
    │     └─ MainWidget->UpdatePlayerHP(0)
    │           └─ 모든 HP 아이콘을 Hidden으로 설정 (0 < 1, 0 < 2, 0 < 3 모두 false)
    │
    └─ [82행] CurHP = MaxHP;  ← CurHP를 3으로 설정하지만 UI 재업데이트 없음!

  4. MainWidget 로직
  void UMainWidget::UpdatePlayerHP(int32 CurHP)
  {
      ImagePlayer00->SetVisibility(ESlateVisibility::Hidden);
      ImagePlayer01->SetVisibility(ESlateVisibility::Hidden);
      ImagePlayer02->SetVisibility(ESlateVisibility::Hidden);

      if (CurHP >= 1)  // 0 >= 1 → false
          ImagePlayer00->SetVisibility(ESlateVisibility::Visible);
      if (CurHP >= 2)  // 0 >= 2 → false
          ImagePlayer01->SetVisibility(ESlateVisibility::Visible);
      if (CurHP >= 3)  // 0 >= 3 → false
          ImagePlayer02->SetVisibility(ESlateVisibility::Visible);
  }

  영향 범위

  즉시 영향:
  - 게임 시작 시 플레이어는 실제로 HP 3을 가지고 있지만 UI에는 0으로 표시됨
  - 플레이어가 "이미 죽은 것처럼" 보여 혼란 초래

  추가 시나리오:
  - 플레이어가 피격당하면 OnDamage() → SetHP() → UpdatePlayerHP() 호출로 UI가 업데이트됨
  - 첫 피격 시에야 비로소 UI가 정상 표시됨 (HP 2개로 표시)
  - 피격 없이 게임을 진행하면 계속 0으로 표시됨

  ---
  수정 방안 (권장)

  위치: PlayerPawn.cpp:77-82

  Before:
  if (AShootingGameMode* GameMode = Cast<AShootingGameMode>(GetWorld()->GetAuthGameMode()))
  {
      GameMode->UpdatePlayerHP(CurHP);  // CurHP = 0
  }
  CurHP = MaxHP;

  After:
  CurHP = MaxHP;  // 먼저 초기화

  if (AShootingGameMode* GameMode = Cast<AShootingGameMode>(GetWorld()->GetAuthGameMode()))
  {
      GameMode->UpdatePlayerHP(CurHP);  // CurHP = 3 전달
  }

  수정 이유:
  1. 가장 간단 - 코드 2줄의 순서만 변경
  2. 로직상 자연스러움 - "값을 먼저 설정하고, UI에 반영"
  3. 성능 영향 없음
  4. 다른 코드에 부작용 없음

  ---
  결론

  원인: CurHP 멤버 변수가 0으로 초기화되어 있고, BeginPlay()에서 UI 업데이트를 CurHP 설정보다 먼저 호출함

  해결: 77-82행의 코드 순서를 변경하여 CurHP = MaxHP 초기화를 UI 업데이트보다 먼저 수행

  우선순위: P1 - 게임 시작 시 플레이어에게 혼란을 주는 UI 버그

  난이도: 매우 낮음 (코드 2줄 순서 변경)
