  2. EnemyBulletActor의 구조적 문제

  PlayerBulletActor (ABulletActor):
  - ABulletActor 클래스 상속
  - DestroyZone과 충돌 시 SetActive(false) + 오브젝트 풀에 반환
  - 17-18개로 일정하게 유지 ✅

  EnemyBulletActor:
  - AActor 직접 상속 (ABulletActor 아님)
  - DestroyZone과 충돌 시 Destroy() 호출되어야 함
  - 8개 → 68개로 계속 증가 ❌

  3. 실제 원인

  적 총알이 증가하는 이유는 2가지 중 하나입니다:

  ① 충돌 프로필 설정 문제 (가장 가능성 높음)

  - EnemyBulletActor는 "EnemyBullet" 프로필 사용
  - DestroyZoneActor는 "DestroyZone" 프로필 사용
  - 두 프로필이 서로 충돌하도록 설정되지 않았을 가능성
  - → DestroyZone과 충돌하지 않아 Destroy()가 호출되지 않음

  ② LifeTime 타이머 미설정

  EnemyBulletActor.cpp:82-84를 보면:
  if (bActive)
  {
      // 비어있음! 여기에 타이머 설정 코드가 없음
  }
  - LifeTime = 5.f 설정은 있지만
  - SetActive(true) 호출 시 타이머를 시작하는 코드가 없음
  - → 5초 후 자동 삭제가 작동하지 않음
