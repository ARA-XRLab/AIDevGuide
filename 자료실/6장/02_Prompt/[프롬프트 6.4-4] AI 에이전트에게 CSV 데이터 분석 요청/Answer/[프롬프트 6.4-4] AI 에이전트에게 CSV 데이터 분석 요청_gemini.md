  2. 근본 원인 특정


   * 최다 호출 이벤트: SS_EnemyMgr_SpawnEnemy (및 이로 인한 SS_Enemy_TickMove)
   * 위치: EnemyManagerActor.cpp 66라인 (스폰 로직 내부)
   * 원인 추론:
       1. 변수 값 오류: EnemyManagerActor.h에 선언된 SpawnFrequency 값이 에디터 혹은 설정 파일에서 0에 가까운 값(예: 0.0001)으로 설정되었을 가능성이 매우
          높습니다.
       2. 이로 인해 타이머가 거의 매 틱마다 수십 번씩 SpawnEnemy를 호출하게 되어, 1초에 수천 마리의 적이 생성되는 "스폰 폭주" 현상이 발생했습니다.
       3. 생성된 적들이 DestroyZone에 도달하기 전까지 모두 Tick을 수행하므로 SS_Enemy_TickMove 시간이 누적되어 전체 프레임 드랍을 유발했습니다.

  3. 구체적인 해결 방법


  A. 변수 수정
   * 대상 파일: Source/SpaceShooter/Public/EnemyManagerActor.h 또는 에디터 내 BP_EnemyManager 인스턴스
   * 수정 변수: SpawnFrequency
   * 적정 값: 1.0f ~ 3.0f (초당 1마리 혹은 그 이하)

  B. 재발 방지를 위한 검증 코드 (Guard Logic)
  SpawnEnemy 함수 내부에 최소 간격 보호 로직과 최대 개수 제한을 추가하여 시스템 안정성을 확보해야 합니다.


    1 // EnemyManagerActor.cpp 수정 예시
    2 void AEnemyManagerActor::SpawnEnemy()
    3 {
    4     // 1. 월드 내 적 액터 수 제한 (과부하 방지)
    5     TArray<AActor*> FoundEnemies;
    6     UGameplayStatics::GetAllActorsOfClass(GetWorld(), AEnemyActor::StaticClass(), FoundEnemies);
    7     if (FoundEnemies.Num() > 100) // 최대 100마리로 제한
    8     {
    9         return; 
   10     }
   11
   12     // 2. SpawnFrequency가 비정상적으로 낮을 경우 강제 조정
   13     if (SpawnFrequency < 0.1f)
   14     {
   15         SpawnFrequency = 0.5f;
   16         // 타이머 재설정 로직 필요
   17     }
   18
   19     // ... 기존 스폰 로직 ...
   20     GetWorld()->SpawnActor<AEnemyActor>(EnemyFactory, SpawnTransform);
   21 }


  요약: 현재 프로젝트는 잘못 설정된 스폰 주기(`SpawnFrequency`)로 인해 초당 수천 개의 액터를 생성하고 있으며, 이것이 CPU 연산과 태스크
  그래프(WaitForTasks)를 마비시키고 있습니다. 해당 변수를 정상 범위(1.0 이상)로 수정하면 즉시 성능이 복구될 것입니다.
